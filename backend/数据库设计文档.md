# PyGrow 数据库设计文档

> 生成日期：2026-06-22 | 数据库：SQLite（sqlite+aiosqlite） | ORM：SQLAlchemy 2.0 异步

---

## 一、核心实体设计

系统共 **6 个核心实体域**，涵盖 **21 张数据表**：

| 实体域 | 包含表 | 说明 |
|--------|--------|------|
| 用户与身份 | `users` | 用户账户、等级、积分 |
| 课程与学习 | `courses`, `lessons`, `course_progress`, `course_collections` | 课程内容、课时管理、学习进度 |
| 练习与测评 | `questions`, `practice_records`, `code_submissions` | 题库、答题记录、代码提交 |
| 游戏化激励 | `badges`, `user_badges`, `daily_tasks`, `user_daily_tasks` | 徽章、每日任务、经验积分 |
| 社区互动 | `posts`, `comments`, `post_likes`, `post_favorites` | 论坛帖子、评论、点赞收藏 |
| 项目挑战 | `projects`, `project_submissions` | 实战项目、提交评审 |

辅助表：`notes`（视频笔记）、`favorites`（通用收藏）、`course_reviews`（课程评价）、`ai_chat_records`（AI 对话记录）。

## 三、实体关系设计

```
                              ┌─────────────────────┐
                              │       users          │
                              │  (用户 — 核心实体)    │
                              └────────┬────────────┘
                                       │ 1
                    ┌──────────────────┼──────────────────────┐
                    │                  │                       │
                   N│                 N│                      N│
         ┌─────────▼──────┐  ┌───────▼────────┐   ┌──────────▼─────────┐
         │ course_progress │  │ practice_records│   │       posts         │
         │  (学习进度)      │  │  (练习记录)      │   │  (论坛帖子)          │
         └────────┬────────┘  └───────┬────────┘   └──────────┬─────────┘
                  │                   │                        │ 1
                  │ 1                 │ N                      ├──────────────────┐
         ┌────────▼──────┐  ┌───────▼────────┐               N│                  │ N
         │    courses     │  │   questions     │   ┌───────────▼──────┐  ┌───────▼────────┐
         │  (课程 — 12门) │  │  (题库 — 2121题) │   │    comments      │  │   post_likes    │
         └────────┬──────┘  └────────────────┘   │  (评论)           │  │  (点赞)          │
                  │ 1                             └──────────────────┘  └────────────────┘
         ┌────────▼──────┐     ┌──────────────────┐
         │    lessons     │     │    badges         │      ┌──────────────────────┐
         │  (课时)         │     │  (徽章定义)        │      │    daily_tasks        │
         └───────────────┘     └────────┬─────────┘      │  (每日任务模板)         │
                                        │ 1                └──────────┬───────────┘
                               ┌────────▼─────────┐                 │ 1
                               │   user_badges     │        ┌────────▼───────────┐
                               │  (用户徽章 N:N)    │        │  user_daily_tasks   │
                               └──────────────────┘        │  (用户任务记录)       │
                                                           └──────────────────────┘
         ┌──────────────────┐     ┌──────────────────┐
         │    projects       │     │   favorites       │      ┌──────────────────────┐
         │  (实战项目)        │     │  (通用收藏 多态)    │      │   ai_chat_records     │
         └────────┬─────────┘     └──────────────────┘      │  (AI 对话历史)         │
                  │ 1                                        └──────────────────────┘
         ┌────────▼─────────┐
         │project_submissions│    ┌──────────────────┐     ┌──────────────────────┐
         │  (项目提交)        │    │    notes          │     │   course_reviews      │
         └──────────────────┘    │  (视频笔记)         │     │  (课程评价)            │
                                 └──────────────────┘     └──────────────────────┘
```

**主要关系说明：**

| 关系 | 类型 | 说明 |
|------|------|------|
| User → CourseProgress | 1:N | 一个用户有多门课程的学习进度 |
| User → PracticeRecord | 1:N | 一个用户有多次练习记录 |
| User → Post / Comment | 1:N | 一个用户可以发多篇帖子和评论 |
| User ↔ Badge (via UserBadge) | N:N | 用户和徽章是多对多，通过关联表实现 |
| User ↔ DailyTask (via UserDailyTask) | N:N | 用户和每日任务是多对多 |
| Course → Lesson | 1:N | 一门课程包含多个课时 |
| Question → PracticeRecord | 1:N | 一道题可以被多次练习 |
| Post → Comment | 1:N | 一篇帖子有多条评论 |
| Project → ProjectSubmission | 1:N | 一个项目可被多人提交 |
| User → Favorite | 1:N | 通用收藏，通过 `item_type` 区分课程/题目 |
| Note → Course / Lesson | N:1 | 笔记关联到课程和具体课时 |

## 四、关键设计说明

### 4.1 段位与经验值系统

- **段位计算**：经验值映射到 8 个段位，算法位于 `app/services/gamification.py`
  - 黑铁 0–99 → 白银 100–299 → 黄金 300–599 → 铂金 600–999 → 钻石 1000–1499 → 星耀 1500–2099 → 王者 2100–2999 → 传奇 3000+
- **积分来源**：完成测评 +20exp，每日一练 +15exp，单题答对 +3exp，完成课程 +10exp
- **段位晋升**：晋级赛（promotion test）≥80% 正确率可升段（入门级→初级→中级→高级）

### 4.2 题库设计

- **6 种题型**：单选、多选、判断、填空、简答、代码
- **难度三级**：easy / medium / hard
- **阶段三级**：初级（12章）、中级（7章）、高级（8章）
- **27 个知识点标签**：从 Python 基础到机器学习/深度学习全覆盖
- **数据来源**：17 个 Excel 文件解析，2,121 道题目
- **判题逻辑**：多选题比较集合（顺序无关），判断题容忍中文/英文/符号多种格式，代码题预留测试用例执行

### 4.3 错题本与智能推荐

- **错题自动入库**：`practice_records.is_correct = False` 自动成为错题
- **收藏即入错题本**：点击爱心收藏时调用 `POST /api/practice/add-wrong` 同步写入
- **智能推荐算法**（`GET /api/practice/recommend`）：
  1. 优先推荐与错题相同知识点标签的题目
  2. 其次推荐相同章节的题目
  3. 然后推荐同阶段、未做过的题目
  4. 无历史记录时均匀采样
