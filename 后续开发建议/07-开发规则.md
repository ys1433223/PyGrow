# Claude Code / Codex 开发规则

## 一、基本要求

请阅读 docs 文件夹下的项目文档，理解本项目是一个名为 PyGrow 的 AI Python 自主学习平台。

请不要一次性实现所有功能。

请按照开发任务拆分文档，从 MVP 阶段开始逐步实现。

每完成一个模块，需要保证：

1. 代码可以运行；
2. 前后端接口能正常通信；
3. 数据库表结构清晰；
4. 没有明显报错；
5. 给出启动方式；
6. 等待用户确认后再继续下一个模块。

---

## 二、技术栈固定

前端：

```text
Vue3 + Vite + Element Plus
后端：
FastAPI
数据库：
MySQL
部署：
Docker Compose + Nginx
AI：
通过环境变量配置 API_KEY
________________________________________
三、开发顺序
请严格按照以下顺序开发：
1. 项目初始化
2. 数据库连接
3. 用户注册登录
4. 课程模块
5. 能力测评
6. 练习模块
7. 积分等级
8. 学习报告
9. AI导师
10. 在线代码运行
11. 项目挑战
12. 社区
13. 后台管理
14. 部署
________________________________________
四、禁止事项
1.	不要一次性生成整个项目；
2.	不要随意更换技术栈；
3.	不要删除已有功能；
4.	不要使用复杂微服务架构；
5.	不要一开始就实现编程农场、宠物养成等扩展功能；
6.	不要在代码中写死 AI API Key；
7.	不要忽略代码运行安全限制。
________________________________________
五、代码风格要求
1.	前端组件命名清晰；
2.	后端接口路径符合 REST 风格；
3.	数据库字段命名统一；
4.	重要函数添加简单注释；
5.	返回数据格式统一；
6.	错误信息要清楚。
________________________________________
六、第一次开发提示词
请复制下面这段给 Claude Code / Codex：
请阅读 docs 文件夹下的项目文档，理解这是一个名为 PyGrow 的 AI Python 自主学习平台。

请不要一次性实现所有功能。

第一阶段只完成项目基础架构和 MVP 核心功能：

1. 前端使用 Vue3 + Vite + Element Plus；
2. 后端使用 FastAPI；
3. 数据库使用 MySQL；
4. 使用 Docker Compose 管理服务；
5. 实现用户注册登录；
6. 实现课程列表与课程详情；
7. 实现能力测评；
8. 实现基础练习题提交；
9. 实现学习进度记录；
10. 实现基础积分等级系统。

请先生成清晰的项目目录结构、数据库表结构和启动方式。
完成后不要继续写其他高级功能，等待我确认。
________________________________________
七、AI导师开发提示词
现在请实现 AI代码导师模块，要求：

1. 后端提供 /api/ai/ask、/api/ai/debug-code 两个接口；
2. 前端提供 AI导师页面；
3. 用户可以输入 Python 代码和问题；
4. 后端调用大模型 API；
5. 返回代码解释、错误原因、修改建议和学习提示；
6. 接口要预留 API_KEY 环境变量配置；
7. AI回答风格适合 Python 初学者，不要直接替学生完成所有作业。
________________________________________
八、在线代码运行开发提示词
现在请实现在线 Python 代码运行模块。

MVP 阶段先使用 FastAPI 后端 subprocess 运行用户代码。

要求：

1. 接口路径为 POST /api/code/run；
2. 输入 code 字符串；
3. 限制运行时间 3 秒；
4. 返回 stdout、stderr、success；
5. 禁止危险导入和文件操作；
6. 后续预留替换为 Docker 沙箱的结构。
________________________________________
九、游戏化成长开发提示词
现在请实现游戏化成长模块。

要求：

1. 用户完成课程、练习、项目后获得经验值；
2. 根据经验值计算段位；
3. 段位包括黑铁、白银、黄金、铂金、钻石、星耀、王者、传奇；
4. 首页展示当前段位和经验进度条；
5. 学习报告页展示积分变化记录。
